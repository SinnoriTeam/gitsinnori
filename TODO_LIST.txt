1. '비동기 메시지' 서버/클라이언트 설계 개선
난이도 : 상
현재 진행 사항 : 없음
문제점 : 싱글 쓰레드 서버 모델이라 읽은 메시지 수 만큼 비지니스 로직을 수행하여 출력 메시지를 버퍼에 저장하여 버퍼 소진
현재 처리 방식 : 서버에서 쓰기 버퍼를 회수하여 일정 버퍼 수 확보 될때까지 읽기 이벤트 처리 중지
할일및 고려사항 : (1-1) 소켓 읽지 않고 쓰기만 하는 클라이언트 인지및 제제
(1-2) 클라이언트 소켓 쓰기/읽기 속도 제어
(1-3) 싱글 쓰레드 서버에서 소켓 쓰기/읽기 속도 제어에 의한 적정 버퍼수 유지

2. UDP 신규 추가
난이도 : 상
현재 진행 사항 : 없음
비지니스 로직 서버와 접속 서버는 UDP 로 메시지 교환하여 효율 증대 필요함

3. 서버 동적 클래스 로더에서 의존성 있는 파일 변경시 비지니스 로직 재 로드
난이도 : 중
현재 진행 사항 : 없음

4. 클라이언트에서 실제 서버 접속하지 않고 가상의 출력 결과를 보낼 수 있는 설계 개선
난이도 : 중
현재 진행 사항 : 없음

5. 서버에서 입력 메시지에 따른 비지니스 로직을 쉽게 교체할 수 있도록 설계 개선
난이도 : 중
현재 진행 사항 : 없음
