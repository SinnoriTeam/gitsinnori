3   서버 구성 요소

3.1   클라이언트 정보 관리자

동기화 하여 신규 클라이언트 접속  승인시 소켓 채널에 자원을 할당하고 소켓이 닫혔을 때에 자원을 해제한다.  소켓 자원은 문자셋, 바이트 오더, 메시지와 이진스트림간의 변환 방법, 읽기 전용 버퍼, 읽기 전용 버퍼 최종 읽은 위치, 최종 읽은 시간, 에코 메시지 보낸 횟수 메일 식별자로 구성된다. 내부 자료 구조를 키 소켓 채널, 값 소켓 자원인 해쉬이다.

3.2   클라이언트 접속 요청 처리기

IP와 port 를 잡고 서버 소켓 채널을 생성시키고 accept 만 허용하는 selector 를 통해서 접속 요청을 하는 소켓 채널이 있다면 접속 요청 큐를 통해서 클라이언트 접속 승인 처리기로 보낸다.

3.3   클라이언트 접속 승인 처리기

폴로 관리되며 접속 요청 큐를 통해서 얻은 소켓 채널 즉, 접속을 요청한 클라이언트를 신규 등록 처리한다.  결과적으로 클라이언트 메시지 읽기 처리기에 1:1로 등록되며 클라이언트 정보 관리자를 통해서 소켓 자원도 할당 받게 된다.

3.4   클라이언트 메시지 수신 처리기

폴로 관리되면 소켓 읽기를 수행하여 데이터 스트림으로 부터 메시지를 추출하여 입력 메시지 큐로 보낸다. 소켓별 수신 버퍼 목록을 가지며 소켓으로 수신된 데이터 스트림은 수신 버퍼 목록에 담긴다. 수신 버퍼 목록 최대 크기는 환경 변수에서 정하며 최소 크기는 1이다. 수신 버퍼 목록은 메시지를 담고 있는 스트림 크기 만큼 커지며 메시지 추출후 버퍼 관리자한테 수신 버퍼를 반환하여 줄어든다.

3.5   입력 메시지 1:1 대응 비지니스 처리기

폴로 관리되며 입력 메시지 큐를 통해 얻은 입력 메시지를 입력으로 비지니스 로직 클래스 관리자를 통해서 얻은 입력 메시지 1:1 대응 비지니스 로직을 수행하여 얻은 출력 메시지들을 출력 메시지 큐로 보낸다. 단 이때 덩치 큰 메시지는 래퍼(=Wrapper) 메시지 그룹에 담고 작은 메시지는 전송(=Carrier) 메시에 담아 보낸다.

3.6   비지니스 로직 클래스 관리자

메시지에 1:1 대응하는 비지지스 로직을 동적 클래스로 로딩하여 요청시 반환한다. 키 메시지 식별자, 값 동적 클래스로 갖는 해쉬로 운영된다. 비지니스 로직 요청시 로딩할 당시 시간과 요청시 해당 클래스 파일의 시간을 비교하여 수정되었다면 재 로딩한후 반환한다.

3.7   클라이언트 메시지 송신 처리기

출력 메시지 큐에서 메시지를 읽어 소켓 쓰기 작업을 수행한다.  이때 소켓 쓰기 작업은 쓰레드간에 경쟁하면서 소켓 쓰기 작업을 수행하기에 동기화를 하면서 수행한다.
