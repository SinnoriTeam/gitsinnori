3.2. 파일 업로드

3.2.1. UpFileInfo

파일 업로드 준비 요청 입력 메시지. 클라인언트는 업로드할 파일 조각을 보낸 준비를 한후 이 메시지를 보낸다. 서버는 이 메시지를 받으면 업로드 파일 조각을 받을 준비를 하고 그 결과를 출력 메시지에 담아 보낸다.

노트:

송수신 과정에서 복사할 원본 파일과 목적지 파일은 다른 프로그램의 간섭을 피하기 위해서 락을 건다. 파일 락은 3가지 경우에 풀린다. (1) 전송이 끝났을 경우 (2) 사용자의 전송 취소 요청시 (3) 전송중 에러 발생시

3.2.2. UpFileInfo.xml

<?xml version="1.0" encoding="UTF-8"?> 

<sinnori_message>

<messageID>UpFileInfo</messageID>

<desc>파일 업로드 준비 요청 입력 메시지</desc>

<singleitem name="localFilePathName" type="ub pascal string">

    <desc>로컬 파일 경로</desc>

</singleitem>

<singleitem name="localFileName" type="ub pascal string">

    <desc>로컬 파일 이름</desc>

</singleitem>

<singleitem name="localFileSize" type="long">

    <desc>로컬 파일 크기</desc>

</singleitem>

<singleitem name="remoteFilePathName" type="ub pascal string">

    <desc>원격지 파일 경로</desc>

</singleitem>

<singleitem name="remoteFileName" type="ub pascal string">

    <desc>원격지 파일 이름</desc>

</singleitem>

<singleitem name="fileBlockSize" type="integer">

    <desc>전송할 파일 조각 크기</desc>

</singleitem>

</sinnori_message>

UpFileInfoResult

노트:

송수신 과정에서 복사할 원본 파일과 목적지 파일은 다른 프로그램의 간섭을 피하기 위해서 락을 건다. 파일 락은 3가지 경우에 풀린다. (1) 전송이 끝났을 경우 (2) 사용자의 전송 취소 요청시 (3) 전송중 에러 발생시
