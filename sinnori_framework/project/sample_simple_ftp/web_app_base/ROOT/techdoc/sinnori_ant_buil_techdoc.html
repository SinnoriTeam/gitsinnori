2. 신놀이 ant 빌드 환경 구축전 고려 사항

a) 신놀이 프로젝트 종속 설정파일 특성

신놀이 프로젝트 종속되는 설정 파일은 환경 변수로 지정되며, 싱글턴 패턴으로 딱 1번 읽어와서 저장된다. 설정파일 내용은 크게 공통 변수와 1개 이상의 프로젝트 변수로 나뉜다. 공통 변수는 광역으로 설정된 설정파일 접근 변수인 conf 를 통해서 접근 가능한 변수를 말한다. 프로젝트 변수는 프로젝트 이름으로 묶이는 클라이언트와 서버 운영에 필요한 변수들의 묶음이다. 프로젝트 변수 갯수는 프로젝트 목록에서 정해진다. 신놀이 응용 프로그램은 여려곳의 서버에 접속 할 수 있도 있거나 혹은 여려 개의 특수 목적을 갖는 서버를 운영할 수 있다. 예를 들면 파일 예제로 제공된 신놀이 파일 송수신 서버에서 신놀이 로그 서버로 연결할 수 있어야 한다. 이때 프로젝트 목록은 "신놀이 파일 송수신", "신놀이 로그" 가 된다.

b) 공통 변수

공통 변수는 광역으로 설정된 설정파일 접근 변수인 conf 를 통해서 접근 가능한 변수를 말한다.

c) 프로젝트 변수

신놀이는 클라이언트 동작을 위해서 필요한 변수들과 서버 운영에 필요한 변수들이 있다. 이를 프로젝트 이름으로 묶어 다루는 변수를 프로젝트 변수라 한다. 프로젝트 변수는 내부적으로 서버/클라이언트 공통 프로젝트, 클라이언트 프로젝트, 서버 프로젝트로 이렇게 3개 부분으로 나뉘어져 있다.

i. 서버/클라이언트 공통 프로젝트 변수

서버/클라이언트 공통적으로 사용하는 변수들로 구성되며, 이 변수를 통해서만 클라이언트 프로젝트 변수와 서버 프로젝트 변수를 얻을 수 있다.

ii. 클라이언트 프로젝트 변수

서버와 접속하기 위한 소켓 폴과 그것을 운영하기 위해 필요한 정보를 가지고 있다. 이 변수는 서버/클라이언트 공통 프로젝트 변수를 통해서만 얻을 수 있다.

iii. 서버 프로젝트 변수

서버를 운영하기 위한 쓰레드 폴들에 필요한 정보를 가지고 있다. 이 변수는 서버/클라이언트 공통 프로젝트 변수를 통해서만 얻을 수 있다.

 

3. jar 파일 특성

jar 파일 만들때 외부 jar 파일을 포함해야 할지 말지 고민이 필요하다. 첫번째 고려사항으로 라이센스가 있다. 예를 들면 대부분의 JDBC 드라이버는 GPL or LGPL 등의 독자 라이센스 때문에 자신의 라이센스와의 궁합을 고려해야 한다. 마지막 두번째 고려사항으로 기술적인 부분이 있다. 미 포함된 jar 파일은 외부 jar 파일과 같이 움직여야 하는 단점이 있다. 반면 모두 포함된 jar 파일은 그렇지 않다.

b) 신놀이 코어 라이브러리

신놀이 프레임워크 핵심 라이브러리이다.

i. 디렉토리 구조

gitsinnori/sinnori_framework/core_build

   .

   |-----build

   |-----dist

   |-----lib

   |-------ex

   |-------in

   |-----src

ii. 주요 디렉토리 설명

1. lib/ex

미 포함될 외부 jar 파일들이 위치한다.

2. lib/in

포함될 외부 jar 파일들이 위치한다.

3. build

소스를 컴파일한 결과 클래스 파일들이 위치하는 경로이다.

4. dist

배포할 jar 파일이 만들어지는 경로

5. build.xml

핵심 라이브러리의 패키지 구성은 공통, 서버, 클라이언트 이렇게 3개 부분으 나뉘어져 있다. 어플이 클라이언 전용만으로 혹은 서버 전용만으로 동작할 수 있다고 보고 서버와 클라이언트 모듈간의 의존성을 없애기 위해서 공통 부분을 컴파일 하고 뒤이어 공통에 종속적인 서버와 클라이언트를 각각 컴파일 한다.

 

c) 프로젝트 - 공통

i. 디렉토리 구조

gitsinnori/sinnori_framework/project/sample_simple_ftp

   .

   |-------rsa_keypair   

   |-------impl

   |---------message

   |-----------info

   |-------log

   |---------apache

   |---------client

   |---------server

   |---------servlet

ii. 주요 디렉토리 설명

1. rsa_keypair

공개키 쌍이 있는 경로

2. impl/messsage/info

메시지를 정의한 xml 파일들이 있는 경로

3. log

log/apache : 우분투의 경우 80포트는 아파치가 선점하고 있다. 그 규칙에 맞추어서 신놀이는 아파치와 톰캣 연동을 시켰다. 아파치와 톰캣 연동시 아파치 부분 설정 파일에서 아파치 로그의 위치가 바로 이 위치가 된다.

log/servlet : 우분투의 경우 80포트는 아파치가 선점하고 있다. 그 규칙에 맞추어서 신놀이는 아파치와 톰캣 연동을 시켰다. 아파치와 톰캣 연동시 톰캣 부분 설정 파일에서 톰캣 로그의 위치가 바로 이 위치가 된다.

log/client : 프로젝트 종속 어플의 로그 위치이다.

log/server : 프로젝트 종속 서버의 로그 위치이다.

 

d) 프로젝트 - 서버

i. 디렉토리 구조

gitsinnori/sinnori_framework/project/sample_simple_ftp/server_build

   .

   |---------build

   |-----------impl

   |-------------executor

   |---------------server

   |---------corelib

   |-----------ex

   |-----------in

   |---------dist

   |-----------lib

   |---------lib

   |-----------ex

   |-----------in

   |---------src

ii. 주요 디렉토리 설명

1. build/impl/executor/server

동적으로 로딩할 서버 비지니스 로직 클래스가 위치한다. 이 위치는 프로젝트의 설정 파일에서 지정되는 위치이다.

2. corelib/ex

포함되지 말아야할 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

3. corelib/in

신놀이 코어 라이브러리 jar 파일과 포함하고자 하는 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

lib/ex

서버 개발에 필요하지만 미 포함되어야할 외부 jar 파일들이 위치한다.

4. lib/in

서버 개발에 필요하고 포함되어야할 외부 jar 파일들이 위치한다.

5. build.xml

신놀이 코어 라이브러리를 컴파일후 만들어지는 jar 파일을 복사해서 컴파일 한다.

 

e) 프로젝트 - 어플리케이션

i. 디렉토리 구조

gitsinnori/sinnori_framework/project/sample_simple_ftp/client_build/app_build

   .

   |-----------build

   |-------------impl

   |---------------executor

   |-----------------client

   |-----------corelib

   |-------------ex

   |-------------in

   |-----------dist

   |-------------lib

   |-----------lib

   |-------------ex

   |-------------in

   |-----------src

ii. 주요 디렉토리 설명

1. build/impl/executor/client

동적으로 호출되는 클라이언트 비지니스 로직 클래스들이 위치하는 경로이다. 프로젝트의 설정파일에서 지정되는 위치이다.

2. corelib/ex

포함되지 말아야할 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

3. corelib/in

신놀이 코어 라이브러리 jar 파일과 포함하고자 하는 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

4. lib/ex

어플 개발에 필요하지만 미 포함되어야할 외부 jar 파일들이 위치한다.

5. lib/in

어플개발에 필요하고 포함되어야할 외부 jar 파일들이 위치한다.

6. build.xml

신놀이 코어 라이브러리를 컴파일후 만들어지는 jar 파일을 복사해서 컴파일 한다.

 

f) 프로젝트 - 웹 어플리케이션

신놀이 웹을 만들기 위해 급조된 구조를 갖고 있다. 웹 어플은 많은 설명이 필요하지만 파일 송수신 2차 버전 개발이 급하므로 생략하겠다.

i. 디렉토리 구조

gitsinnori/sinnori_framework/project/sample_simple_ftp/client_build/web_build

   .

   |-----------build

   |-------------webdistlib

   |-------------weblib

   |-----------corelib

   |-------------ex

   |-------------in

   |-----------dist

   |-----------lib

   |-------------ex

   |-------------in

   |-----------src

   |-------config

ii. 주요 디렉토리 설명

1. corelib/ex

포함되지 말아야할 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

2. corelib/in

신놀이 코어 라이브러리 jar 파일과 포함하고자 하는 신놀이 코어 라이브러리가 의존하는 외부 jar 파일들이 위치한다.

3. lib/ex

웹 어플 개발에 필요하지만 미 포함되어야할 외부 jar 파일들이 위치한다.

4. lib/in

웹 어플 개발에 필요하고 포함되어야할 외부 jar 파일들이 위치한다.

5. build.xml

신놀이 코어 라이브러리를 컴파일후 만들어지는 jar 파일을 복사해서 컴파일 한다.

 

g) 프로젝트 - 웹 루트

톰캣 루트 디렉토리이다. 웹 어플과 마찬가지로 신놀이 웹을 만들기 위해 급조된 구조를 갖고 있다. 웹 어플은 많은 설명이 필요하지만 파일 송수신 2차 버전 개발이 급하므로 생략하겠다.

i. 디렉토리 구조

gitsinnori/sinnori_framework/project/sample_simple_ftp/web_app_base/ROOT

   .

   |-----------WEB-INF

   |-------------classes

   |---------------kr

   |-----------------pe

   |-------------------sinnori

   |---------------------common

   |-----------------------javabeans

   |-------------lib

ii. 주요 디렉토리 설명

1. WEB-INF/classes

서블릿과 "웹 라이브러리 클래스"들이 위치한다.

2. WEB-INF/classes/kr/pe/sinnori/common/javabeans

자바 빈즈들이 위치하는 경로

3. WEB-INF/lib

웹 라이브러리 jar 파일들이 위치하는 경로
